# Firmware

How the NordicTrack S22i custom firmware works, what it does, and what's left to consider.

## Overview

A single OTA zip replaces the stock firmware with a rooted, re-signed build that installs Hyperborea as a privileged system app and demotes iFit's ERU to a powerless state. The OTA is built by `repack_ota.py` from the stock OTA plus our modifications, signed with our OTA key, and deployed via `deploy_ota.sh`.

| RAM / Storage | 2GB DDR3 / 8GB eMMC |
| Display | 1920x1080 landscape, 22", 160 DPI |

### Partition Layout

| Partition | Device | Size | Notes |
|-----------|--------|------|-------|
| Boot | mmcblk0p1 | 64MB | Patched ramdisk |
| System | mmcblk0p2 | 2GB | ~1.1GB free |
| Recovery | mmcblk0p6 | 64MB | Signing key patched by deploy script |
| Backup | mmcblk0p9 | 1.5GB | Factory restore image |
| Data | mmcblk0p10 | 3GB | User data, packages.xml |

## What the OTA Does

### Boot Image (ramdisk patches via `binpatch_ramdisk.py`)

Binary-patches 4 properties in the ramdisk (never extract+rebuild — preserves 300,000+ cpio inodes):

| Property | Stock | Patched |
|----------|-------|---------|
| `ro.secure` | 1 | 0 |
| `ro.adb.secure` | 1 | 0 |
| `ro.debuggable` | 0 | 1 |
| `persist.sys.usb.config` | none | adb |

Kernel cmdline adds: `buildvariant=userdebug androidboot.selinux=permissive`

### System Partition

| Change | Purpose |
|--------|---------|
| **35 APKs re-signed** with our platform key | All system apps match our cert |
| **ERU demoted** to `/system/app/`, `sharedUserId` stripped, signed with throwaway key | Loses all signature/privileged permissions |
| **ERU manifest stripped** of crashing receivers (`TabletStartupReceiver`, `UsbDeviceAttachedReceiver`) | Prevents crash-loops from missing permissions |
| **Hyperborea** installed to `/system/priv-app/` with platform key | UID 1000, SYSTEM + PRIVILEGED |
| **`build.prop`** patched | `persist.sys.usb.config=adb`, `ro.adb.secure=0`, keeps `ro.build.type=user` for stealth |
| **`otacerts.zip`** replaced with our OTA signing cert | Framework can verify our OTAs |
| **`install-recovery.sh`** replaced | Prevents stock recovery restore; handles post-wipe UID fixup |
| **iFit Launcher** HOME intent removed, re-signed | Stock launcher becomes default |

### Data Partition (via `ota_postinstall`)

A statically-linked ARM64 binary runs in recovery (no shell available) and handles:

1. **`.wolfDev`** — creates `/sdcard/.wolfDev` so ERU preserves ADB/dev mode
2. **ADB keys** — pre-authorizes the build machine's ADB public key
3. **Conditional `packages.xml` wipe** — first install only (marker file `/data/system/.hyperborea`); needed because platform key change requires PM to rebuild from scratch
4. **UID fixup marker** — creates `/data/system/.hyperborea_fixup` when wiping, signals `install-recovery.sh` to fix data directory ownership on first boot

### Recovery Partition (via `deploy_ota.sh`, not baked into OTA)

The deploy script patches the recovery signing key on-device before triggering the OTA install. This can't be in the OTA itself because recovery must already trust our key to run the OTA.

## Platform Key Architecture

Three keys generated by `generate_keys.py`:

| Key | Signs | Purpose |
|-----|-------|---------|
| **Platform** | System APKs + Hyperborea | Hyperborea gets `sharedUserId=android.uid.system` (UID 1000) and all signature-level permissions |
| **OTA** | The OTA zip itself | Recovery verifies this before applying |
| **Throwaway** | ERU only | Ensures ERU's cert doesn't match platform, stripping signature permissions |

Result:

| App | UID | Location | Permissions |
|-----|-----|----------|-------------|
| Hyperborea | 1000 (system) | `/system/priv-app/` | All signature + signatureOrSystem |
| ERU | Own UID (10048) | `/system/app/` | No signature permissions, no `INSTALL_PACKAGES` |

ERU is kept (not removed) because it downloads GlassOS ecosystem apps. But demoted, it cannot silently install them — it must use `ACTION_INSTALL_PACKAGE` which requires user interaction.

## Protection Model

With ERU demoted, most of the old defense layers are unnecessary. ERU cannot:
- Disable apps (`pm disable` requires signature permissions)
- Change secure settings (`WRITE_SECURE_SETTINGS` not granted)
- Install apps silently (`INSTALL_PACKAGES` not granted)
- Trigger OTA installs (`RECOVERY` permission not granted, recovery requires our signing key)
- Access USB as system (`MANAGE_USB` not granted)

What remains:

| Protection | Mechanism |
|------------|-----------|
| **`.wolfDev`** | ERU checks for this on boot — preserves ADB and dev mode |
| **`install-recovery.sh`** | Replaces stock script that would restore iFit's recovery image. Also handles post-wipe UID fixup on first boot. |
| **Recovery signing key** | Recovery only accepts OTAs signed with our key |
| **Platform key** | Only APKs signed with our key get system permissions |

## First Boot vs Subsequent Updates

### First Install (platform key change)

1. `ota_postinstall` finds no marker → wipes `packages.xml` so PM rebuilds with our certs
2. PM assigns new UIDs to user-installed apps (may differ from old UIDs)
3. `install-recovery.sh` detects fixup marker → waits for PM, then chowns all `/data/data/*` dirs to match new UIDs
4. Fixup marker is deleted; install marker is created

### Subsequent OTA Updates

1. `ota_postinstall` finds marker → skips `packages.xml` wipe
2. UIDs are preserved, app data survives intact
3. No fixup needed

## Build & Deploy

### Prerequisites

```bash
yay -S android-apktool android-ndk    # ERU decompilation + ota_postinstall compilation
python3 iFit/firmware/tools/generate_keys.py   # One-time: generate all signing keys
```

### Build

```bash
./gradlew :app:assembleSystemDebug              # Hyperborea APK (platform-signed)
python3 iFit/firmware/tools/repack_ota.py       # Repack OTA (re-sign, demote ERU, add Hyperborea)
```

### Deploy (full OTA)

```bash
./iFit/firmware/tools/deploy_ota.sh iFit/firmware/repack/MGA1_20210901/output/MGA1_20210901-signed.zip
```

The deploy script handles: device discovery, recovery key patching, OTA push, install trigger, reboot wait, and 16-point verification.

### Deploy (app-only update)

```bash
./gradlew :app:assembleSystemDebug
adb push app/build/outputs/apk/system/debug/app-system-debug.apk /system/priv-app/Hyperborea/Hyperborea.apk
adb shell chmod 644 /system/priv-app/Hyperborea/Hyperborea.apk
adb reboot
```

The system flavor signs with the platform key automatically — no manual re-signing needed.

### Recompile `ota_postinstall`

After modifying `ota_postinstall.c`:

```bash
/opt/android-ndk/toolchains/llvm/prebuilt/linux-x86_64/bin/aarch64-linux-android21-clang \
    -static -o iFit/firmware/tools/ota_postinstall_arm64 \
    iFit/firmware/tools/ota_postinstall.c
```

## Recovery Procedures

### After Software Factory Reset (Settings menu)

Wipes `/data` + `/cache` only. System and boot partitions survive. Hyperborea is still installed.

```bash
adb root
adb shell touch /sdcard/.wolfDev    # Restore .wolfDev
```

### After Hardware Factory Reset (hold Power during boot)

Restores ALL partitions from backup. Everything is gone — full redeploy required:

```bash
./iFit/firmware/tools/deploy_ota.sh iFit/firmware/repack/MGA1_20210901/output/MGA1_20210901-signed.zip
```

### If Keys Are Regenerated

Delete the install marker so the next OTA wipe packages.xml again:

```bash
adb shell rm /data/system/.hyperborea
```

## Critical Rules (from 25 install attempts)

| Rule | Why |
|------|-----|
| Use `binpatch_ramdisk.py` for ramdisk | Only method that preserves cpio inode structure |
| Pad with `\n` not `\0` | Null bytes kill Android init's property parser |
| Use Python gzip, not macOS gzip | macOS gzip produces different output → wrong boot.img size → hang |
| `buildvariant=userdebug` in cmdline | Required for `ALLOW_ADBD_NO_AUTH=1` |
| Keep `ro.build.type=user` | Stealth — apps see stock device |
| Strip comments from recovery `/res/keys` | `load_keys()` fails if first char isn't `v` |
| Recovery key: curly-brace format | `v2 {64,0xN,...}` — space-separated decimal does NOT work |
| Patch recovery on-device, not macOS | Avoids macOS cpio bugs |

## Still to Consider

- **In-app OTA updates**: The framework now has our real OTA cert in `otacerts.zip`. If we implement standard `RecoverySystem.installPackage()` updates, Android will verify the signature natively. Need to decide: OTA-based system updates vs APK-only pushes.
- **Software factory reset recovery**: `.wolfDev` is lost on factory reset (lives on `/data`), but ERU no longer has `WRITE_SECURE_SETTINGS` so it can't actually disable ADB anyway. The file is a safety net (ERU skips the attempt if it exists), not a critical defense. Low priority.
- **ERU crash behavior**: ERU runs fine with demoted permissions, but some features (like app installs) will silently fail or show unexpected UI. Monitor for user-facing issues.
- **Backup partition**: The factory restore image (mmcblk0p9) still contains stock firmware. A hardware reset undoes everything. Could overwrite with our image, but that removes the safety net.
